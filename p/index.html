QT JSON 完整学习笔记（含字节/字符串/网络传输）

一、QT JSON 核心类（基础必掌握）

类名 核心作用 常用方法/注意事项
QJsonDocument JSON 数据的“容器+转换器”，连接 JSON 字节流与 QT 内部 JSON 对象/数组 - toJson()：转 QByteArray（支持 Compact/Indented 格式）
- fromJson()：从字节流解析，需传入 QJsonParseError
QJsonObject 对应 JSON 对象（{}），存储键值对（键=字符串，值=QJsonValue） - insert(key, value)：添加键值对
- contains(key)：检查键是否存在
- [key]：取值（需先做类型检查）
QJsonArray 对应 JSON 数组（[]），存储有序的 QJsonValue 列表 - append(value)：添加元素
- size()：获取长度
- 遍历：for 循环/迭代器
QJsonValue 封装所有 JSON 基础类型（字符串/数字/布尔/对象/数组/null） - 类型判断：isString()/isDouble()/isArray()/isObject()
- 类型转换：toString()/toInt()/toArray()
QJsonParseError 存储 JSON 解析错误信息（原因、位置） - error：错误类型（NoError 表示无错）
- errorString()：错误描述
- offset：错误位置

// 1. 构建JSON对象
QJsonObject userObj;
userObj.insert("name", "张三");    // 字符串值
userObj.insert("age", 25);         // 数字值（自动转QJsonValue）
userObj.insert("isStudent", false);// 布尔值

// 2. 构建JSON数组并嵌套
QJsonArray hobbyArr;
hobbyArr.append("编程");

// 3. 封装为QJsonDocument，转字节流（QByteArray）
QJsonDocument doc(userObj);
QByteArray jsonBytes = doc.toJson(QJsonDocument::Compact); // 紧凑格式（传输用）
// QByteArray jsonBytes = doc.toJson(QJsonDocument::Indented); // 格式化（调试用）

QJsonParseError parseError;
// 1. 从字节流解析为QJsonDocument（必传错误对象）
QJsonDocument parseDoc = QJsonDocument::fromJson(jsonBytes, &parseError);

// 2. 检查解析错误（第一步必做）
if (parseError.error != QJsonParseError::NoError) {
    qDebug() << "解析失败：" << parseError.errorString() << "位置：" << parseError.offset;
    return;
}

// 3. 检查类型并提取数据（必做类型检查）
if (parseDoc.isObject()) {
    QJsonObject parseObj = parseDoc.object();
    // 读取普通值
    if (parseObj.contains("name") && parseObj["name"].isString()) {
        QString name = parseObj["name"].toString();
    }
    if (parseObj.contains("age") && parseObj["age"].isDouble()) { // JSON数字统一存为double
        int age = parseObj["age"].toInt();
    }
    // 读取数组
    if (parseObj.contains("hobbies") && parseObj["hobbies"].isArray()) {
        QJsonArray hobbies = parseObj["hobbies"].toArray();
        for (auto val : hobbies) {
            if (val.isString()) qDebug() << val.toString();
        }
    }
}

三、JSON 与字节/字符串的转换（关键关联）

1. 核心逻辑

JSON 本质是“UTF-8 编码的文本字节流”，QT 中以 QByteArray 存储，与 QString 转换需指定 UTF-8 编码（避免乱码）。

2. 转换示例

// 1. JSON → QString（用于界面显示/编辑）
QByteArray jsonBytes = doc.toJson();
QString jsonStr = QString::fromUtf8(jsonBytes); // 字节→字符串（UTF-8）

// 2. QString → JSON（从界面输入解析）
QByteArray bytesFromStr = jsonStr.toUtf8(); // 字符串→字节（UTF-8）
QJsonDocument parseDoc = QJsonDocument::fromJson(bytesFromStr, &parseError);

3. 关联知识点补充

类型 本质与作用 核心区别
QByteArray 字节数组（存储原始二进制/字节流），无编码概念 用于 JSON 传输/存储，是 QT JSON 转换的核心载体
QString Unicode 文本字符串（UTF-16 编码），专门处理多语言文本 用于界面显示，与 QByteArray 转换必须指定 UTF-8 编码
char 单个8位字节（基础类型），char[] 是静态字节数组（需手动管理内存） QByteArray 是封装后的“智能字节数组”，自动扩容/内存管理，更安全便捷
二进制 字节的底层存储形式（1字节=8位0/1），输出需手动拆解位（无原生输出方法） qDebug()/std::cout 会默认解读为字符/数字，需函数拆解才能显示二进制串

// .pro 文件需添加：QT += network
#include <QTcpSocket>

QTcpSocket socket;
socket.connectToHost("127.0.0.1", 8888); // 连接服务端

// 连接成功后发送JSON
connect(&socket, &QTcpSocket::connected, [&]() {
    QJsonObject obj;
    obj.insert("name", "张三");
    obj.insert("age", 25);
    // 转紧凑格式字节流（减少传输体积）
    QByteArray jsonBytes = QJsonDocument(obj).toJson(QJsonDocument::Compact);
    socket.write(jsonBytes); // 发送字节流
});

// 接收服务端回复
connect(&socket, &QTcpSocket::readyRead, [&]() {
    QByteArray respBytes = socket.readAll();
    QJsonDocument respDoc = QJsonDocument::fromJson(respBytes);
    // 解析回复...
});

2. 跨语言服务端（通用逻辑）

QT 客户端发送的是标准 UTF-8 编码 JSON 字节流，无 QT 专属格式，任何语言服务端都可处理：

服务端语言 核心流程
C 原生 接收字节流 → 用 cJSON 库解析 → UTF-8 解码
Python data = conn.recv(1024) → json.loads(data.decode('utf-8'))
Java InputStreamReader(in, "UTF-8") → JsonReader 解析

3. 关键注意事项

  • 传输格式：用 Compact 紧凑格式（减少体积）；
  • 编码统一：必须用 UTF-8 编码（避免中文乱码）；
  • 粘包处理：约定分隔符（如 \n）或“先传长度再传数据”，避免解析错误。

五、核心避坑点与最佳实践

  1. 类型检查是刚需：
    • 取值前必须用 contains() 检查键存在，用 isString()/isArray() 检查类型；
    • JSON 数字在 QT 中统一存为 double，需用 toInt()/toDouble() 转换。
  2. 编码不能错：
    • QByteArray ↔ QString 必须用 fromUtf8()/toUtf8()，否则中文乱码。
  3. 错误处理不能少：
    • 解析 JSON 必须传入 QJsonParseError，捕获格式错误；
    • 网络传输需处理连接失败、断开等异常。
  4. 跨语言兼容：
    • 服务端无需依赖 QT，只需按 UTF-8 解码、标准 JSON 解析即可。

六、核心知识点总结

  1. QT JSON 核心是 QJsonDocument（转换）、QJsonObject/QJsonArray（数据结构）、QJsonValue（值封装）、QJsonParseError（错误处理）；
  2. JSON 操作核心流程：构建对象/数组 → 转字节流（QByteArray），或从字节流解析 → 类型检查后取值；
  3. 字节/字符串转换、网络传输的核心是“UTF-8 编码统一”，跨语言通信的关键是“标准 JSON 格式”。
