一、信号的核心特点

特点 说明
异步 信号的到来与进程的执行流无关，可随时打断当前代码。
信息量少 信号本身只携带一个整数编号（事件类型），无法附带字符串或复杂数据。
默认动作 每个信号有系统预设的默认行为（终止、忽略、暂停、继续等）。
可捕获/忽略 除 SIGKILL 和 SIGSTOP 外，进程可以改变信号的处理方式。
不可靠性 普通信号（非实时）可能丢失（多个相同信号未处理时会被合并为一次）。
实时信号 SIGRTMIN ～ SIGRTMAX 支持排队，并可附带一个整数/指针（通过 sigqueue）。

二、常见标准信号（非实时）

信号名 编号 (x86_64) 默认动作 典型触发场景
SIGINT 2 终止进程 用户按 Ctrl+C
SIGQUIT 3 终止 + core dump 用户按 Ctrl+\
SIGKILL 9 终止（不可捕获） kill -9 <pid>
SIGTERM 15 终止（可捕获） kill <pid>，请求进程优雅退出
SIGSTOP 17,19,23 暂停进程（不可捕获） kill -STOP <pid>
SIGCONT 18,19,25 继续运行 kill -CONT <pid>
SIGSEGV 11 终止 + core dump 非法内存访问（段错误）
SIGPIPE 13 终止进程 向已关闭读端的管道写入数据
SIGALRM 14 终止进程 alarm() 定时器超时
SIGCHLD 17 忽略 子进程状态变化（退出、暂停等）
SIGUSR1 / SIGUSR2 10,12 终止进程 用户自定义用途

注意：编号可能因架构略有不同（如 ARM、x86_64），建议使用宏名而非数字。

三、典型应用场景

  1. 进程控制

    • 优雅停止：SIGTERM（可捕获，清理资源后退出）
    • 强制杀死：SIGKILL（无法捕获）
    • 暂停/继续：SIGSTOP / SIGCONT

  2. 异常通知

    • 子进程退出：父进程捕获 SIGCHLD，调用 wait() 回收，避免僵尸进程。
    • 管道破裂：捕获 SIGPIPE，防止程序意外终止。

  3. 定时器

    • 使用 alarm() + SIGALRM 实现简单超时控制。

  4. 用户自定义通信

    • 用 SIGUSR1 / SIGUSR2 或实时信号通知进程执行某个动作（如重新加载配置）。

  5. 守护进程（daemon）重载配置

    • 许多服务（如 nginx、httpd）收到 SIGHUP 后重新读取配置文件。

// catch_sigint.c
#include <stdio.h>
#include <signal.h>
#include <unistd.h>

void handler(int sig) {
    // 注意：printf 不是异步信号安全的，仅用于示例
    printf("捕获到信号 %d，但我不会退出！\n", sig);
}

int main() {
    signal(SIGINT, handler);

    printf("进程 PID = %d，等待 Ctrl+C 或 kill -2 ...\n", getpid());
    while (1) pause();
    return 0;
}

// send_signal.c
#include <stdio.h>
#include <signal.h>
#include <unistd.h>
#include <sys/wait.h>

void child_handler(int sig) {
    if (sig == SIGUSR1) {
        printf("子进程：收到 SIGUSR1，执行自定义动作（如重载配置）。\n");
    }
}

int main() {
    pid_t pid = fork();

    if (pid == 0) {
        signal(SIGUSR1, child_handler);
        printf("子进程 PID = %d，等待信号...\n", getpid());
        while (1) pause();
    }
    else if (pid > 0) {
        sleep(1);
        printf("父进程：发送 SIGUSR1 给子进程 %d\n", pid);
        kill(pid, SIGUSR1);
        sleep(1);
        kill(pid, SIGTERM);
        wait(NULL);
    }
    return 0;
}

// sigaction_demo.c
#include <stdio.h>
#include <signal.h>
#include <unistd.h>
#include <string.h>

void handler(int sig, siginfo_t *info, void *context) {
    printf("收到信号 %d，发送方 PID = %d\n", sig, info->si_pid);
}

int main() {
    struct sigaction sa;
    memset(&sa, 0, sizeof(sa));
    sa.sa_sigaction = handler;
    sa.sa_flags = SA_SIGINFO;

    sigaction(SIGUSR1, &sa, NULL);

    printf("进程 PID = %d，发送 SIGUSR1 给自己...\n", getpid());
    kill(getpid(), SIGUSR1);
    pause();
    return 0;
}

#include <stdio.h>
#include <signal.h>
#include <unistd.h>

void rt_handler(int sig, siginfo_t *info, void *ctx) {
    printf("收到实时信号 %d，附加值 = %d\n",
            sig, info->si_value.sival_int);
}

int main() {
    struct sigaction sa = { .sa_sigaction = rt_handler, .sa_flags = SA_SIGINFO };
    sigaction(SIGRTMIN, &sa, NULL);

    printf("接收端 PID = %d\n", getpid());
    while (1) pause();
}

#include <signal.h>
#include <stdio.h>
#include <stdlib.h>

int main(int argc, char *argv[]) {
    if (argc != 2) {
        fprintf(stderr, "用法: %s <目标PID>\n", argv[0]);
        return 1;
    }
    pid_t target = atoi(argv[1]);
    union sigval val;
    val.sival_int = 42;
    sigqueue(target, SIGRTMIN, val);
    printf("发送实时信号 SIGRTMIN 给 %d，附带值 %d\n", target, val.sival_int);
    return 0;
}

#include <stdio.h>
#include <signal.h>
#include <unistd.h>

void alarm_handler(int sig) {
    printf("定时器超时！\n");
}

int main() {
    signal(SIGALRM, alarm_handler);
    alarm(3);
    printf("等待3秒...\n");
    pause();
    return 0;
}

五、重要注意事项（避坑指南）

  1. 信号处理函数中只能调用异步信号安全的函数

    • 安全函数：write(), _exit(), sigaction(), sleep()（部分）
    • 不安全函数：printf(), malloc(), fopen() 等（示例中为简单用了 printf，实际生产应避免）。
    • 若必须打印，可改用 write(STDOUT_FILENO, "msg\n", len)。

  2. signal() 在不同平台行为不一致

    • 某些旧系统处理后会将处理函数重置为默认。
    • 推荐始终使用 sigaction()。

  3. SIGKILL 和 SIGSTOP 无法被捕获、忽略或阻塞。

  4. 信号会打断部分系统调用（如 read, write, sleep）

    • 默认返回 -1，errno = EINTR。
    • 可通过 sigaction 设置 SA_RESTART 标志让内核自动重启这些调用。

  5. 普通信号（1～31）可能丢失

    • 若需要可靠传递多个事件，请使用实时信号（SIGRTMIN ～ SIGRTMAX）。
