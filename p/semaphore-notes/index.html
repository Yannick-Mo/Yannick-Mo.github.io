信号量（Semaphore）学习笔记

全面梳理信号量核心概念：P/V 操作、POSIX 命名信号量、System V 信号量、多元信号量场景、死锁与异常安全，包含完整代码示例和常见陷阱说明

2026-06-13
阅读时长: 6 分钟

定位：信号量是一种同步原语，不传递业务数据，只传递"许可"或"条件满足"的信号。它需要与共享资源（共享内存、文件、设备等）配合使用，负责控制访问顺序与并发数量。

1. 核心概念

  • 本质：由内核维护的非负整数计数器
  • 核心能力：
    • P 操作（Wait/减1）：若计数器 > 0 则原子减 1 并立即返回；若为 0 则阻塞，直到其他进程执行 V 操作
    • V 操作（Post/加1）：原子加 1；若有进程正在等待，唤醒其中一个
  • 原子性保证：所有 P/V 操作都是内核保证的原子操作，多核环境下安全
  • 无所有权：任何进程都可以 post，这和互斥锁（通常由持有者解锁）不同
  • 类型：
    • 二元信号量（Binary Semaphore）：仅取值 0/1，行为类似互斥锁（Mutex），但通常没有"所有权"概念
    • 计数信号量（Counting Semaphore）：初值可大于 1，用于管理一组同质资源或控制并发数

2. PV 操作详解 

1
2

P (Proberen, 尝试/等待)   →  sem_wait()  或  semop(-1)
V (Verhogen, 增加/释放)  →  sem_post() 或  semop(+1)

  • P 操作逻辑（原子执行）：

    1. 如果 value > 0，value-- 并立即返回
    2. 如果 value == 0，调用进程进入阻塞队列，等待被唤醒

  • V 操作逻辑（原子执行）：

    1. value++
    2. 如果等待队列非空，唤醒其中一个进程

  • 非阻塞 P：sem_trywait() – 值>0 时减1，否则返回错误而非阻塞

  • 超时 P：sem_timedwait() – 指定绝对超时时间

  • 读取当前值：sem_getvalue() – 非原子，仅用于调试，不可用于同步逻辑

3. 标准 C++ 线程信号量 vs 进程间信号量

机制 是否可用于进程间？ 说明
std::counting_semaphore (C++20) 否（默认） 标准库用于线程同步，位于进程私有内存，不可跨进程
std::binary_semaphore (C++20) 否（默认） 同上
POSIX 命名信号量 (sem_open) ✅ 内核持久化，通过名称全局共享，推荐
POSIX 匿名信号量 (sem_init) ⚠️ 有条件 若 pshared != 0 且置于共享内存中，可用于 fork 后的父子进程
System V 信号量 (semget) ✅ 通过 key 标识，功能强大但接口复杂，支持信号量集与 UNDO
Windows 信号量对象 ✅ 内核对象，跨进程，API 风格不同，本文不展开

结论：C++ 进程间通信用POSIX 命名信号量最直接，可移植性最好。

#include <semaphore.h>
#include <fcntl.h>
#include <sys/stat.h>

// 打开或创建
sem_t *sem_open(const char *name, int oflag, mode_t mode, unsigned int value);
// 关闭（进程退出时自动关闭）
int sem_close(sem_t *sem);
// 删除名字（所有引用关闭后内核销毁）
int sem_unlink(const char *name);

// P 操作
int sem_wait(sem_t *sem);
int sem_trywait(sem_t *sem);
int sem_timedwait(sem_t *sem, const struct timespec *abs_timeout);

// V 操作
int sem_post(sem_t *sem);

// 查询值（非原子，仅调试）
int sem_getvalue(sem_t *sem, int *sval);

4.2 命名规则

  • 名称必须以 / 开头，如 /myapp_sem
  • 实际在系统中会映射为文件（如 Linux 下 /dev/shm/sem.xxx）
  • 长度限制一般 NAME_MAX - 4

4.3 使用方法模板

创建/初始化（仅一方创建，避免竞态）：

sem_t *full  = sem_open("/full", O_CREAT, 0644, 0);

// 生产者

// ... 写入共享内存（需额外互斥锁保护缓冲区内部分配） ...
sem_post(full);

// 消费者
sem_wait(full);
// ... 读取共享内存 ...
sem_post(empty);

4.4 清理

  • 所有进程都 sem_close 后，再调用 sem_unlink 彻底删除（类似文件 unlink 语义）
  • 一般由创建者在退出前负责 sem_unlink，或在 atexit 中处理

5. System V 信号量（备选方案）

接口复杂，但支持操作一组信号量、SEM_UNDO等高级特性。

#include <sys/sem.h>

// 创建/获取信号量集，返回 ID
int semget(key_t key, int nsems, int semflg);

// 控制：SETVAL/GETVAL/删除等
int semctl(int semid, int semnum, int cmd, ...);

// 执行 P/V 操作（可同时操作多个信号量）
int semop(int semid, struct sembuf *sops, size_t nsops);

struct sembuf {
    unsigned short sem_num;  // 集合中的索引
    short          sem_op;   // 操作：-1(P) 或 +1(V)
    short          sem_flg;  // IPC_NOWAIT, SEM_UNDO
};

5.2 特点

  • 信号量集：一次 semop 可原子地对多个信号量操作，适合复杂资源分配（如哲学家问题）
  • SEM_UNDO：进程退出时内核自动撤销该进程对信号量的操作，防止死锁（异常安全）
  • 持久化：与 POSIX 类似，信号量集生命期与内核相关，需显式删除

5.3 初始化陷阱

System V 没有原子的创建+初始化步骤，常见做法：

  • 一个进程用 O_CREAT | O_EXCL 创建并初始化
  • 其他进程直接获取并等待初始化完成（例如通过另一个信号量或轮询 semctl 的 GETVAL 等待非零值）

semop(semid, &v, 1);

// 删除
semctl(semid, 0, IPC_RMID);

6. 多元信号量（计数信号量）的典型场景

初值 N > 1，管理资源池或并发数量上限：

  1. 限制并发进程数：如最多允许 5 个计算进程同时运行，信号量初值 = 5，每进程进入前 wait，退出后 post
  2. 连接池：最多 10 个数据库连接，信号量初值 = 10，控制同时使用数
  3. 硬件资源：3 台打印机，初值 = 3
  4. 有界缓冲区：空位信号量初值 = 缓冲区大小，满信号量初值 = 0
  5. 流量整形：控制请求速率（令牌桶算法可用信号量实现）

7. 常见问题与必要补充知识

7.1 死锁与异常安全

  • 死锁风险：多个信号量时，不同进程的 P 操作顺序不一致会死锁。保持全局一致的加锁顺序。
  • 进程崩溃：持有信号量（P 后未 V）的进程崩溃，可能导致其他进程永久阻塞。
    • POSIX：无自动恢复机制，可结合 sem_timedwait 超时处理，或使用心跳/监视进程。
    • System V：可使用 SEM_UNDO，内核会在进程退出时回滚操作。

7.2 初始化竞态

  • POSIX 使用 O_CREAT | O_EXCL 保证只有一方创建并初始化。
  • 切勿让多个进程随意 sem_unlink 然后重新创建，否则状态丢失。

7.3 命名冲突

  • 全局名称空间，不同应用应使用唯一前缀，如 /myapp_sem_empty

7.4 信号量并非可重入锁

  • 二元信号量不同于 PTHREAD_MUTEX_RECURSIVE，同一进程对同一个信号量连续两次 P（且中间无 V）会死锁。

7.5 与条件变量的比较

特性 信号量 条件变量
状态 基于整数计数 基于谓词 + 互斥锁保护
唤醒丢失 无丢失（V 会递增计数） 可能丢失（signal 时无人等待）
跨进程 可以 通常仅限线程（Pthread）
生产者-消费者 简洁，天然计数 需自行维护计数和判断

7.6 性能考量

  • 每一次 sem_wait/sem_post 都是系统调用，频繁操作可能成为瓶颈；极高频率同步可考虑共享内存 + 原子变量 + 自旋 + futex（仅 Linux），复杂度显著上升。

8. 一句话总结

信号量 = 原子计数器 + 阻塞队列
用 P 等待资源，用 V 释放资源；它不传输数据，而是控制"何时能访问共享资源"的规则。进程间通信推荐使用 POSIX 命名信号量；复杂场景（需事务性操作多个信号量或进程退出自动恢复）可选用 System V。
